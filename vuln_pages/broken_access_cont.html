What?

Broken Access Control

Broken access control occurs when a user is able to act outside of their intended permissions. This can include accessing resources they should not see or modifying aspects not they are not authorized to.


Why?

This is one of the most common web application vulnerabilities and is consistently ranked among the top risks by OWASP. It can lead to unauthorized access to sensitive information as well as data modification, creation, or deletion.


How?

This vulnerability occurs when systems fail to properly enforce access restrictions on users. In order to prevent this, applications should validate permissions on the server side and ensure users can only access resources they are authorized to use. A proper example includes, Just-in-Time (JIT) permissions, where a user is only granted permissions at that moment for a limited time.
