What?

Insecure Design

Insecure design refers to flaws in an application's architecture or overall design that fail to consider security from the beginning. These issues often arise when systems are built without proper planning for how users should interact with them securely.


Why?

When security is not considered during the design phase, vulnerabilities can, unknowingly, be built directly into the system. This can expose sensitive information or allow unintended actions, even if later protections are added. Issues such as poor access flow, lack of validation, or missing safeguards can make systems easier for attackers to exploit.


How?

This vulnerability occurs when applications are designed without incorporating security principles, such as validating user actions or restricting system behavior. To prevent this, developers should follow secure design practices, including threat modeling and planning for how users and data should be handled safely.
